## 前言

[Claude Code](https://claude.com/claude-code) 是 Anthropic 推出的终端原生 AI 编程助手。除了在交互中自然描述需求之外，它还提供了一套以 `/` 开头的**斜杠命令（Slash Commands）**，用于执行配置、会话管理、代码审查、项目初始化等操作。

本文基于实际使用经验，对所有内置斜杠命令进行系统梳理。

---

## 1. 会话管理

控制当前对话会话的状态与生命周期。

| 命令 | 功能 | 示例 / 说明 |
|------|------|-------------|
| `/clear` | 清空当前会话上下文 | 开始全新对话，不影响项目文件和设置。适合切换话题或上下文过长时使用 |
| `/compact` | 压缩会话上下文 | 将对话历史总结为摘要以释放 token 空间。压缩前确保重要内容已落地到文件中 |
| `/rename` | 重命名当前会话标题 | `/rename "修复登录页面布局问题"` |
| `/resume` | 恢复之前的会话 | 继续之前的对话 |
| `/export` | 导出当前会话记录 | 支持多种格式，用于存档、分享或审计 |
| `/prompt` | 显示当前系统提示词 | 查看 CLAUDE.md 注入的项目上下文、记忆、环境信息等 |

## 2. 配置与偏好

调整 Claude Code 的外观、行为和模型偏好。

| 命令 | 功能 | 示例 / 说明 |
|------|------|-------------|
| `/config` | 打开设置界面 | 修改主题、模型、权限策略、hooks 等。等价于编辑 `~/.claude/settings.json` 和 `.claude/settings.json` |
| `/theme` | 切换终端颜色主题 | 暗色 / 亮色 / 系统跟随 |
| `/statusline` | 配置状态栏显示项 | 选择显示模型名、分支、token 用量等信息 |
| `/model` | 切换 AI 模型 | 可用模型取决于订阅计划和最新模型列表 |
| `/fast` | 切换 Fast 模式 | 使用 Opus 模型并加速输出，适合需要快速响应的场景 |

## 3. 项目初始化与记忆

| 命令 | 功能 | 示例 / 说明 |
|------|------|-------------|
| `/init` | 生成项目的 CLAUDE.md | 分析项目结构，自动生成包含技术栈、目录结构、常用命令的文档。建议每个项目根目录运行一次 |
| `/memory` | 管理持久记忆 | `/memory 记住：这个项目使用 Biome 格式化代码`。记忆存储在 `.claude/projects/<path>/memory/` |
| `/recall` | 检索相关记忆 | `/recall 代码格式化工具`。Claude Code 启动时自动加载 `MEMORY.md` 索引 |

## 4. 代码审查与质量

| 命令 | 功能 | 示例 / 说明 |
|------|------|-------------|
| `/review` | 审查当前 Pull Request | `/review --effort high` 指定审查强度；覆盖正确性、安全性、性能、可维护性 |
| `/security-review` | 安全漏洞专项审查 | 覆盖 OWASP Top 10：注入、认证绕过、敏感信息泄露、不安全反序列化等 |
| `/simplify` | 审查并自动简化代码 | 识别可简化、复用、提效的代码模式并自动修复，侧重质量而非 bug |
| `/code-review` | 分级代码审查 | `/code-review --effort low`（仅高置信度）/ `max`（全面）/ `--comment`（发布到 PR） |

## 12. 实用技巧

### 12.1 命令组合

多个命令可以串联达到高效 workflow：先 `/review` 审查，再 `/simplify` 自动修复；先 `/init` 初始化，再 `/config` 自定义。

### 12.2 善用 `/prompt` 排查问题

当 Claude Code 的行为不符合预期时，用 `/prompt` 查看它实际「看到」了哪些指令——包括 CLAUDE.md、memory、系统提示词等。

### 12.3 `/compact` 的时机

当出现以下情况时建议执行压缩：

- 上下文 token 使用率超过 80%
- 对话中已完成多个独立任务
- 响应速度明显下降

### 12.4 项目级 vs 用户级配置

| 级别 | 路径 | 作用范围 |
|------|------|----------|
| 用户级 | `~/.claude/settings.json` | 适用于所有项目的全局设置 |
| 项目级 | `.claude/settings.json` | 仅对当前项目生效，覆盖用户级设置 |

使用 `/config` 编辑时注意区分两者的作用范围。

### 12.5 Memory 的最佳实践

- 记录项目特有的约定和非标准做法
- 记录用户偏好（语言、格式化风格等）
- 避免记录可从代码或 git 历史自动推导的信息
- 定期用 `/recall` 检查记忆的有效性

## 小结

Claude Code 的斜杠命令体系涵盖了从会话管理、代码审查、配置调整到自动化编排的完整开发工作流。掌握这些命令可以大幅提升你在终端中的开发效率——将 AI 从「对话伙伴」升级为「可编排的编程环境」。

核心原则：**能说的就直接说，需要精确控制时用命令**。大部分日常操作通过自然语言即可完成，斜杠命令用于配置固化、会话管理、多步骤编排等需要明确控制的场景。
